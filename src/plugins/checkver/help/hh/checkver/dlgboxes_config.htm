Sally: Plugins: Check Version: Dialog Boxes

Configuration of Check Version

You can customize behaviour of the Check Version plugin in this dialog. See Configuring Plugins for general information about configuration.

Options

Open Check for New Versions window on Sally start
Choose the period for automatic checking for newer stable Sally releases on GitHub (never, every day, every week, every month, every three months, or every six months).
When Check for New Versions window is opened, hide it and perform auto-connect
When checked, the plugin automatically connects to GitHub Releases and gets information about the latest stable release. Otherwise you must click on the Check Now button in the Check For New Versions window to start checking.
Close window if no new version is found, otherwise show window
When checked, the plugin is quiet until it finds a newer stable release. If a new release is available, it shows the Check For New Versions window with found updates. When not checked, you must close the Check For New Versions window manually every time using the Close button.
Set defaults
Choose this button to restore the default automatic-check schedule and window behavior.
